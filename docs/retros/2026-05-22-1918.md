## Learning Candidate

**Pattern then Prevention:**

> Pattern: When a bug-class appears in one file, the same class of bug often lives in N other files in the same architecture (today: hash-mismatch in guardrail-state, then read-state, then gaming-detector, three iterations of the same fix shape). Iterating one-by-one wasted cycles AND the partial state between iterations triggered compounding false-positives elsewhere (the SKIP_PATTERNS narrowing in session-audit.ts blocked the v3 push because gaming-detector.mjs still had the old list).
>
> Prevention: When fixing a bug-class, grep ALL files in the same architecture for the pattern BEFORE applying the first fix. Apply as one batch in one commit. Today the pattern recurred 3 times across two repos because I treated each instance as a one-off.
